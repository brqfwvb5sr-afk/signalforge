# Plan

## Current Goal

Choose and build a programming project that is fun, public, and large enough for roughly 20-30 hours of work.

## Default Milestones

1. Pick the final project idea.
2. Create the MVP spec and acceptance criteria.
3. Build the core engine or core app flow.
4. Add the first usable UI.
5. Add tests for the risky logic.
6. Add polish, content, and deployment.
7. Review, debug, and publish.

## Candidate Project

Signalforge: a browser-based automation puzzle game where players program small robots on a grid to move resources, operate machines, and solve levels.

## Next Decision

Confirm whether Signalforge is the chosen project or replace it with a different idea before implementation starts.
